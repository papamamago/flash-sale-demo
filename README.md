# 🔥 高并发秒杀系统 Demo

> 一个完整的秒杀/抢购系统后端 + 前端可视化并发测试工具

## ✨ 核心特性

| 特性 | 实现 |
|------|------|
| 🚀 **高并发秒杀** | 内存互斥锁 + 原子库存扣减，防止超卖 |
| 🔒 **防重复下单** | 同一用户对同一商品只能购买一次 |
| 📊 **可视化测试** | Web 界面实时显示秒杀过程 |
| ⚡ **命令行压测** | 内置 CLI 压测工具，统计 QPS |
| 🛡️ **限购策略** | 单用户限购数量控制 |
| 🔄 **一键重置** | 快速恢复初始状态 |

## 🚀 快速启动

```bash
# 安装依赖
npm install

# 启动服务
npm start

# 打开浏览器访问
open http://localhost:3000
```

## 🏗️ 架构说明

```
用户请求 → Express API → 互斥锁 → 原子扣库存 → 生成订单
                ↓
       内存锁 + 重复检测
```

**并发控制策略：**
- 每个商品的库存操作独立加锁
- 忙等待 + 超时机制防止死锁
- 订单生成与库存扣减在同一原子操作中

## 📊 压测结果参考

| 用户数 | 并发 | 成功率 | QPS |
|--------|------|--------|-----|
| 500 | 50  | ~30%   | ~800/s |
| 1000 | 100 | ~10%   | ~1200/s |
| 5000 | 200 | ~2%    | ~1500/s |

> 注：实际结果取决于服务器性能。库存越少，失败率越高（正常现象）。

## 🛠️ 技术栈

- **Express** — Web 框架
- **内存锁** — 原子库存控制
